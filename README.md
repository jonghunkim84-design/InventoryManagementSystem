# 스마트 재고 관리 AI 시스템

소상공인 F&B 매장을 위한 스마트 재고 관리 시스템입니다.  
OCR 기반 자동 입고, BOM 기반 자동 소진, 실시간 로스 관리를 지원합니다.

---

## 주요 기능

| 기능 | 설명 |
|---|---|
| **자동 입고** | 영수증 스캔 데이터를 AI 매칭으로 식재료와 연결, 단위 자동 환산 후 재고 증가 |
| **BOM 기반 자동 소진** | POS 판매 데이터 입력 시 레시피(BOM) 기준으로 식재료 재고 자동 차감 |
| **로스(폐기) 관리** | 유통기한 경과·변질 등 폐기 사유 기록 및 재고 차감 |
| **실사 보정** | 월말 재고 실사 결과로 재고 수량 보정 및 조정 이력 기록 |
| **재고 부족 알림** | 안전재고 미달 품목 자동 탐지 및 권장 발주량 추천 |
| **대시보드 API** | 재고 현황, 폐기 누계, 입고 누계 통합 데이터 제공 |

---

## 기술 스택

- **언어**: Python 3.10+
- **DB**: SQLite
- **AI 매칭**: difflib (유사도 기반 퍼지 매칭)

## 파일 구조

```
InventoryManagementSystem/
├── database.py          # DB 초기화, 커넥션 컨텍스트 매니저
├── ocr_matcher.py       # AI 매칭 엔진 (부분 문자열 + difflib 퍼지 매칭)
├── incoming_service.py  # 입고 처리 서비스 (단위 환산, 재고 증가)
├── sales_service.py     # 판매 소진 서비스 (BOM 기반 재고 차감)
├── alert_service.py     # 재고 부족 탐지 및 발주 추천
├── advanced_features.py # 폐기 처리, 실사 보정, 대시보드 데이터 API
├── schema.sql           # 기본 DB 스키마
├── patch_schema.sql     # 추가 테이블 스키마 (waste_logs, inventory_adjustments)
└── main.py              # 전체 플로우 시연 엔트리포인트
```

## 핵심 로직

### OCR 매칭 (ocr_matcher.py)

영수증에서 스캔된 식재료명을 DB 등록 식재료와 연결합니다.

1. **부분 문자열 매칭** (1순위): 등록명이 스캔명 안에 포함되면 즉시 매칭  
   예) `'깐마늘'` → `'마늘'` 포함 → 마늘로 매칭
2. **difflib 퍼지 매칭** (2순위): 유사도 점수 기반 최근접 매칭

### 단위 환산 (ingredient_conversions)

입고 단위와 재고 기본 단위가 다를 때 자동 환산합니다.

| 입고 단위 | 환산 | 기본 단위 |
|---|---|---|
| 망 | × 5,000 | g |
| 통 | × 1,000 | g |
| 단 | × 200 | g |
| 근 | × 600 | g |

### 재고 부족 알림

`current_stock <= safety_stock` 조건으로 미달 품목을 탐지하며,  
권장 발주량은 `안전재고 × 2`로 산정합니다.
